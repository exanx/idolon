Self-Hosting Guide

Deploy your own private instance of Idolon using Firebase and Cloudflare. Complete control, zero cost, total privacy.

Overview Prerequisites 1. Firebase Setup 2. Secure Proxy 3. Frontend Config 4. Deployment

Why Self-Host?

Idolon is designed as a "Bring Your Own Key" (BYOK) application. By self-hosting, you ensure that:

  • You own the infrastructure (Firebase & Cloudflare).
  • You control the API keys entirely.
  • You eliminate any dependency on the official hosted version.
  • You can modify the code, add features, or change themes.

Prerequisites

Before starting, ensure you have the following accounts and tools installed:

Accounts

  • GitHub (to clone code)
  • Google Firebase (Spark Plan - Free)
  • Cloudflare (Free Plan)

Tools (CLI)

  • Node.js (v18+)
  • npm install -g firebase-tools
  • npm install -g wrangler
1

Firebase Setup (Backend)

Firebase handles authentication and encrypted database storage for chat history.

1. Create Project

Go to the Firebase Console, create a new project (e.g., idolon-my-instance). Disable Google Analytics to keep it simple.

2. Enable Authentication

Navigate to Build → Authentication. Click "Get Started" and enable:

  • Google (No setup required usually)
  • Email/Password

3. Create Firestore Database

Navigate to Build → Firestore Database. Click "Create Database".

  • Select Production Mode.
  • Choose a location close to you (e.g., nam5 (us-central)).

4. Set Security Rules

Go to the Rules tab in Firestore. These rules ensure users can only access their own data. Replace everything with:

firestore.rules 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Helper function to check ownership
    function isOwner(userId) {
      return request.auth != null && request.auth.uid == userId;
    }

    // Match users collection
    match /users/{userId} {
      // Allow user to read/write their own user document
      allow read, write: if isOwner(userId);
      
      // Match subcollections (settings, chats, secrets)
      match /{document=**} {
        allow read, write: if isOwner(userId);
      }
    }
  }
}

5. Get Config

Go to Project Settings (Gear icon). Scroll down to "Your apps", select the Web icon (</>), register the app (no hosting yet), and copy the firebaseConfig object. You will need this for Step 3.

2

Secure Proxy (Cloudflare)

To safely use API keys in a web app, we use a Cloudflare Worker as a proxy. This hides your keys from the network tab if you implement server-side storage, but mainly it acts as a secure relay.

1. Initialize Worker

Terminal 
npm create cloudflare@latest idolon-proxy
# Select "Hello World" Worker
# Select "No" to TypeScript (for simplicity)
cd idolon-proxy

2. Configure Wrangler

Edit wrangler.toml to set compatibility:

wrangler.toml 
name = "idolon-proxy"
main = "src/index.js"
compatibility_date = "2024-01-01"
node_compat = true

3. Deploy Proxy Logic

Since the full proxy source code is extensive (handling Gemini, Groq, etc.), for a personal instance where you trust yourself, you can technically skip the proxy and put keys directly in the client settings. However, to enable the "Secure Proxy" feature in Idolon, simply create the worker and note the URL.

(Refer to the source code repository for the `proxy/index.js` file content).

4. Deploy

Terminal 
npx wrangler deploy

Copy the URL provided (e.g., https://idolon-proxy.username.workers.dev).

Frontend Configuration

Now, connect the code to your new backend.

1. Edit app.js

Open app.js in your code editor. Look for the top section.

    authDomain: "idolon-my-instance.firebaseapp.com",
    projectId: "idolon-my-instance",
    storageBucket: "...",
    messagingSenderId: "...",
    appId: "..."
};

// 2. REPLACE THE PROXY URL FROM STEP 2 (Optional)

const proxyUrl = 'https://idolon-proxy.YOUR-NAME.workers.dev'; // NEW

// 3. REMOVE THE GATEKEEPER
// Scroll to the bottom of app.js.

Important: Ensure you delete the "Gatekeeper" code block at the very bottom of `app.js` or the app will refuse to load on your own domain!
4

Deployment

You can host the frontend anywhere (Vercel, Netlify, GitHub Pages), but since you are already using Firebase, it's easiest to host it there.

1. Initialize Hosting

Terminal (Project Root) 
firebase init hosting

# 1. Select "Use an existing project" -> Choose your project
# 2. What do you want to use as your public directory? -> . (current directory)
# 3. Configure as a single-page app? -> No
# 4. Set up automatic builds with GitHub? -> No
# 5. File index.html already exists. Overwrite? -> No

2. Deploy

Terminal 
firebase deploy --only hosting

Success! Firebase will give you a Hosting URL (e.g., https://idolon-my-instance.web.app). Open it, sign in, and enjoy your private AI chat.
